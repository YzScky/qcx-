# QCX 币安合约自动交易系统

**纯评分驱动 — 无需人工判断，数据规则说了算**

一个全自动的币安 USDⓈ-M 合约交易机器人。只做多，动态杠杆，硬止损保护。所有决策都是数据驱动的硬规则，无人工干预、无主观判断。

## 功能特点

- **纯评分驱动**：无 BTC 硬性限制（仅作信息参考），无日亏损限制，完全信任评分系统
- **一级过滤**：入场时机评分（≥12 分才开仓，不再有 BTC 大盘过滤层）
- **选币排序**：资金费率(空头挤压) + 成交量爆发 + 多周期共振 + 涨幅区间
  - 只排序，不作为过滤条件 — 所有候选币都走入场时机判断
- **入场时机独立判断**：价格位置 + 15m趋势 + 止跌信号 + 量能恢复
  - 超买惩罚：15m阳线≥80%扣15分，≥70%扣8分 — 防止连续阳线追高
- **三级仓位**：≥75分→10x/95u(满仓)，60-74→5x/55u(半仓)，40-59→3x/30u(轻仓)
- **智能止盈持仓**：K线特征区分正常回调 vs 趋势走坏
  - 有下影线 + 缩量 + 近支撑 + 大周期向上 → 正常回调，继续持有
  - 不满足的 → tp 或 sl 出场
- **趋势加速止损**：1h连6阴→sl+25，1h连4阴→sl+12，1h连3阴+15m也向下→sl+8，5m连5阴→sl+15
- **硬止损保护**：亏损≥-7%立即平仓
- **最大6个持仓**：只做多

## 安装

```bash
# 1. 配置 Binance API
mkdir -p ~/.binance
cat > ~/.binance/trading << 'EOF'
api-key=YOUR_API_KEY
api-secret=YOUR_SECRET_KEY
env=prod
EOF

cat > ~/.binance/active_profile << 'EOF'
name=trading
EOF

# 2. 配置代理（如在中国大陆）
# SOCKS5 代理在 127.0.0.1:7891（RabbitPro/Clash）

# 3. 设置定时任务（每3分钟运行一次）
cronjob action=create \
  name="币安全自动交易系统" \
  prompt='运行 `python3 ~/.hermes/scripts/binance_strategy.py` 并输出结果' \
  schedule="every 3m" \
  repeat=9999
```

## 评分体系

### 📡 选币排序（scan_market）

**仅用于排序候选池，不当作过滤条件。** 高分币按顺序走入场时机判断。

| 维度 | 最高分 | 说明 |
|:---|:---:|:---|
| 资金费率 | 20 | 负费率 = 空头挤压潜力，散户空头过多可能触发轧空 |
| 成交量爆发 | 15 | 近1h成交量 vs 前3h均值，突然放量 = 主力入场可能 |
| 多周期共振 | 15 | 15m+1h同时翻多+10，4h也向上再+5 |
| 涨幅区间 | 10 | 8-20%=10(刚启动)，20-30%=5(追高风险)，30-60%=2(近见顶) |

### 🎯 入场时机（entry_timing）— 满分30分

**独立于选币评分** — 高分币出来后独立判断入场点。

| 维度 | 最高分 | 说明 |
|:---|:---:|:---|
| 价格位置 | 0~8 | 在15m区间所处位置，低位高分；>70%高位 → 禁止开仓 |
| 15m短期趋势 | 0~4 | 近期K线方向 |
| 止跌信号 | 0~3 | 下影线≥实体2倍 = 下方承接 |
| 缩量回调 | 0~3 | 成交量萎缩 |
| 1h趋势 | 0~5 | 1h方向向上加分 |
| 4h趋势 | 0~5 | 4h方向向上加分 |
| 连阴惩罚 | 0~-5 | 连续3阴以上扣分 |
| 放量下跌惩罚 | 0~-5 | 量比>1.5意味着抛压 |
| 超买惩罚 | -8~-15 | 15m阳线≥80%→-15，≥70%→-8 |

### 💎 持仓评估（analyze_position）

**每3分钟评估每笔持仓**，用 K 线特征区分正常回调 vs 趋势走坏。

| 特征判断 | 决策 |
|:---|:---:|
| 有下影线 + 缩量 + 近支撑 + 大周期向上 | ✅ 持有（正常回调） |
| 不满足上述条件 | ❌ 止盈 或 止损 |
| 1h连6阴 | ❌ 止损（sl+25，强制） |
| 1h连4阴 | ❌ 止损（sl+12） |
| 1h连3阴+15m也向下 | ❌ 止损（sl+8） |
| 5m连5阴 | ❌ 止损（sl+15，加速下跌） |

**决策逻辑：** `sl >= hold → 止损 / tp > hold → 止盈 / hold 最高 → 持有`

- 固定止盈（原8%/15%）已移除，完全由评分系统动态决定
- 评分说"止盈"但PnL为负 → 标记为"止损(趋势走弱)"
- 所有判断都基于K线指标（下影线、量比、支撑距离、连续阴线数），无硬阈值

## 注意事项

- 需要 SOCKS5 代理（中国大陆访问 Binance API）
- API Key 需开通合约交易权限（仅需可读+开平仓权限）
- 系统自动运行，无需人工干预
- 所有API密钥存放在 `~/.binance/trading`，不提交到仓库
- 每3分钟运行一次，输出结果到 cron 日志
